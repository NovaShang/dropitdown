# DropItDown 产品需求文档 (PRD)

## 1. 产品定位

面向 Mac 用户的拖拽式 AI 文件归档工具。

- **痛点**：日常碎文件（截图、账单、文档）命名混乱、无处安放；存档后变"黑盒"，未来检索困难。
- **方案**：拖文件到图标 → MarkItDown 清洗为 Markdown → LLM 对比目录树智能分类 → 原文件与 MD 笔记双重落档。

---

## 2. 核心工作流

1. **触发**：拖文件到 Dock 图标，App 唤醒并加入处理队列（Dock 出现运行小点）。
2. **清洗**：调用 MarkItDown 转出 Markdown 文本（图片走多模态 LLM 描述，音频走转写后端）。
3. **分类**：将文本片段 + 当前目录树喂给 LLM，决定归档路径并生成一句话摘要。
4. **归档**：
   - 原文件移到 `[原文件根目录]/[AI判定路径]/`
   - MD 笔记写到 `[MD根目录]/[AI判定路径]/`
5. **通知**：系统通知本次归档结果，点击可打开主界面查看。

---

## 3. 功能设计

### 3.1 生命周期

**退出条件（统一规则）**：主界面未打开 **且** 没有进行中的任务，立即退出。

由此推导出两种典型场景：

- **拖拽流**：拖入文件唤醒，处理队列运转期间不退出；队列清空时若主界面未打开则立即退出。
- **管理流**：用户点击 Dock 图标，或处理过程中点击图标打开主界面 → 主界面常驻。用户关闭主界面时，若仍有任务在跑则等任务完成后退出，否则立即退出。

**首次运行**：引导用户将 App pin 到 Dock，并完成基础配置。

### 3.2 配置项

- **原文件根目录**：物理仓库（如 `~/Documents/Archive`）。
- **Markdown 根目录**：笔记仓库（可指向 Obsidian Vault）。
- **LLM 后端**（可分别配置）：
   - 文本分类 LLM（OpenAI / Claude / 本地 Ollama）
   - 图片描述 LLM（多模态模型）
   - 音频转写后端（Whisper API / Azure Speech / `speech_recognition` 默认）
- **目录扫描**：扫描两个根目录的文件夹结构作为分类 context，自动忽略噪音目录（`node_modules`、`venv`、`.git`、`.next`、`dist`、`__pycache__` 等，列表可编辑）。

### 3.3 归档与转换规则

- **MD 文件结构**：顶部 YAML 元数据 + 原文件链接：

   ```markdown
   ---
   original_file: "file:///Users/username/Documents/Archive/Finance/2026_AWS.pdf"
   archived_at: 2026-05-28
   summary: "2026年5月份的AWS账单，总计128.5美元。"
   ---
   # 原始文件内容...
   ```

- **智能路径匹配**：LLM 命中已有目录则直接放入；识别为新类型则在双端根目录同步新建子目录。

### 3.4 历史记录面板（管理流）

- **时间线**：倒序展示归档记录：原文件名、AI 摘要、目标路径。
- **直达**：原文件名 → Show in Finder；MD 图标 → 系统默认编辑器打开。
- **逐条撤销**：每条记录提供撤销按钮，将原文件移回原位置。MD 笔记不参与回滚，保留为孤儿笔记并在 UI 上标注，可手动删除。
- **检索**：搜索框模糊匹配历史记录的文件名 / 摘要 / 路径。

---

## 4. 非功能需求

1. **本地优先**：文件位移、目录扫描全部本地完成；LLM 调用按用户配置可云可本地，密钥仅本地存储。
2. **超大文件成本控制**：对超长 MarkItDown 输出做截断或采样后再喂分类 LLM，避免单次调用成本失控（截断策略可配）。
3. **撤销安全**：归档操作记录在本地日志中，历史面板可随时撤销。若原位置已被外部修改/占用，标记为"不可安全撤销"并提示用户。
